01. Prime Value Generation

Value in this system is not arbitrary; it is derived from the Prime Number Sequence. Every unit of work performed by a Node advances its internal counter. This counter corresponds to an ordinal position in the sequence of prime numbers.

Work (Counts) → Ordinal Index → Prime Value (Scarcity)

02. Heuristic Currency

The system operates on a 5-tier liquidity spectrum based on time and geometry. Each unit acts as a Precedent for value calculation.

QUADRANT (◴) The Base Unit. Represents 1/4 of a cycle. High scarcity threshold.
DAY (☼) Standard operational unit. Moderate scarcity.
DEGREE (°) Geometric unit. 1/360th of a cycle.
TWIN (♊) Rare prime alignment. Extremely high value density.

03. Thermodynamic Entropy

The system enforces thermodynamic equilibrium through Entropy. Transferring value creates an energy differential.

  • Positive Entropy: Generated when value flows "downhill" (Rich → Poor). This expands the system surplus.
  • Negative Entropy: Generated when value flows "uphill" (Poor → Rich). This consumes system energy and can lead to Void events (deflation).

04. Rolling Momentum

Unlike traditional proof-of-work where progress is binary, this engine uses Rolling Momentum. When a Node mints a Book, it only spends the exact "Counts" required for that difficulty tier. Any excess work performed is preserved, carrying over to the next cycle. Work is never wasted.
